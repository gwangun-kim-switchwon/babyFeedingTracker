## 2026-03-31: 첫 번째 사용 피드백 반영

### 수정 1: 경과 시간 "전" 분리 표시 → 한 줄 통합

**문제:** "2시간 15분" 다음 줄에 "전"이 별도 Text로 표시됨.

**원인:** 디자인 시 시각적 계층(큰 글씨 + 작은 보조 글씨)을 의식해서 "전"을 분리했지만, 실제로 써보니 의미 단위가 쪼개져서 한눈에 안 들어옴.

**교훈:** 짧은 정보는 한 줄로 압축하는 것이 기본. 줄을 나누려면 "나눠서 더 읽기 쉬운가?"를 먼저 검증. 특히 핵심 숫자 정보("2시간 15분 전")는 쪼개면 안 된다.

**수정:** `formatElapsedTimeDisplay()` 반환값에 "전"을 포함시키고, 별도 Text composable 삭제.

### 수정 2: 수유 기록 추가 후 바텀시트 수동 오픈 → 자동 오픈

**문제:** 기록 추가 후 리스트에 항목이 추가되지만, 분유/모유 선택을 위해 항목을 탭해야 한다는 걸 사용자가 모름.

**원인:** 개발자 관점에서는 "탭하면 열린다"가 당연하지만, 실제 사용자는 리스트 항목이 클릭 가능하다는 힌트가 없으면 발견하지 못함.

**교훈:** 숨겨진 인터랙션(hidden affordance)에 의존하지 않는다. 다음 행동이 명확한 경우(기록 직후 → 종류 선택) 자동으로 다음 UI를 띄워준다.

**수정:** `addRecord()` 후 ViewModel이 새 레코드를 StateFlow로 emit, UI가 관찰하여 바텀시트 자동 오픈.

---

### 수정 3: 바텀시트 수동 닫기 → 선택 시 자동 닫힘 (신규 기록만)

**문제:** 모유/분유를 선택해도 바텀시트가 열려 있어서 매번 뒤로가기나 바깥 탭으로 닫아야 함.

**원인:** 편집과 신규 입력의 UX를 동일하게 처리함. 편집은 여러 필드를 바꿀 수 있어야 하지만, 신규 입력은 빠르게 끝나야 함.

**교훈:** 반복 빈도가 높은 행위(수유 기록은 하루 수십 번)는 탭 수를 극단적으로 줄여야 한다. 신규 입력과 편집의 UX를 분리하라: 신규는 최소 탭 + 자동 닫힘, 편집은 자유롭게 열어둠.

**수정:** `isNewRecord` 플래그로 모드 구분. 신규: 모유 선택 → 즉시 닫힘, 분유 → 용량 선택 → 즉시 닫힘. 편집: 자동 닫힘 없음.

## 패턴 정리

| 패턴 | 규칙 | 근거 |
|------|------|------|
| 정보 압축 | 짧은 정보는 한 줄 | 모바일에서 줄 쪼개기 = 공간 낭비 + 인지 부하 |
| 자동 유도 | 다음 행동이 자명하면 자동 오픈 | 숨겨진 인터랙션은 발견되지 않음 |
| 모드 분리 | 신규 입력 ≠ 편집 | 반복 행위는 최소 탭, 편집은 자유도 |
| 시각적 연결 | 관련 항목에 타임라인/연결선 | 단순 나열보다 맥락 전달력 높음 |
